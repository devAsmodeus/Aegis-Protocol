# Days 3-5 Implementation Plan

> **For agentic workers:** Use superpowers:executing-plans to implement task-by-task. Steps use `- [ ]` syntax for tracking.

**Goal:** Land Day 3 (RAG service), Day 4 (Agent runtime), Day 5 (Channel adapters) per `docs/AI_USAGE.md` roadmap. One PR per day. Branch off `main` each time. Merge via `gh pr merge --squash --delete-branch`.

**Architecture:**
- Day 3 wraps the existing `aegis/retrieval/` hybrid pipeline in a tenant-scoped `aegis/rag/service.py` and adds an optional FlashRank reranker adapter behind the existing `Reranker` Protocol.
- Day 4 introduces `aegis/agent/` with an async tool-loop (`Runtime`), `LLMClient` Protocol + deterministic stub, `Tool` Protocol + initial RAG tool, and a `ReceiptSink` Protocol with a SQLAlchemy implementation.
- Day 5 introduces `aegis/channels/` with a `ChannelAdapter` Protocol, an in-memory test adapter, and Telegram (AIOgram) + Discord (discord.py) thin adapters that translate platform events into agent calls.

**Tech Stack:** Python 3.12, FastAPI, async SQLAlchemy 2, asyncpg, Qdrant, FastEmbed, FlashRank (optional), AIOgram, discord.py, structlog. uv + ruff + mypy strict + pytest.

**Hard rules (from CLAUDE.md):**
- Tests requiring docker services → `@pytest.mark.integration`. Default CI green without docker.
- Run `uv run pytest -m "not integration"`, `uv run ruff check . && uv run ruff format --check .`, `uv run mypy aegis` before each commit.
- Update `docs/AI_USAGE.md` Day-N entry per new top-level module.
- ENS / 0G / channels secrets stay out of the repo. Channel tokens come from `Settings`.
- New ORM models: `sqlalchemy.Uuid` PK + `TimestampMixin`. Migrations are append-only.
- Receipts store **content-hashes**, not Qdrant point IDs.

- [ ] Implement `RagService` dataclass(slots=True):
  ```python
  @dataclass(slots=True)
  class RagService:
      pipeline: HybridPipeline

      async def search(
          self,
          *,
          text: str,
          tenant_id: str | None = None,
          filters: dict[str, Any] | None = None,
          k: int | None = None,
      ) -> list[RetrievalHit]:
          query = RetrievalQuery(text=text, tenant_id=tenant_id, filters=filters or {})
          if k is not None:
              # one-off override: return top-k after pipeline produces final_k
              hits = await self.pipeline.retrieve(query)
              return hits[:k]
          return await self.pipeline.retrieve(query)
  ```

### Task 4.3: `Runtime.run` tool-loop

**Files:**
- Create: `aegis/agent/runtime.py`
- Create: `tests/test_agent_runtime.py`.

- [ ] `Runtime(llm, tools, receipt_sink, max_tool_calls=4)`.
- [ ] Loop:
  1. Build messages list from `AgentRequest`.
  2. Call `llm.complete(messages, tools)`.
  3. If `text`: collect retrieval-hashes from prior tool results, ask sink to record, return `AgentResponse`.
  4. If `tool_calls`: dispatch, append result to messages, increment counter; on overflow break with `text="(tool budget exhausted)"`.
- [ ] Tests cover: pure-text response, single tool call → text, tool-call overflow path, missing-tool error path (raises `UnknownToolError`).
- [ ] Receipt: input_hash = sha256(request.text), output_hash = sha256(response.text), retrieval_ids = collected content-hashes, tools_used = ordered tuple.
